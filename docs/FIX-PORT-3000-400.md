# Fix 400 on http://nexlify.live:3000

The error **"The plain HTTP request was sent to HTTPS port"** means nginx on the VPS still has `listen 3000 ssl`. The repo fix was never applied on the server.

## One command on the VPS

```bash
cd /path/to/nexlify-panel
sed -i 's/\r$//' scripts/vps-fix-400-port-3000.sh
sudo bash scripts/vps-fix-400-port-3000.sh
```

## If you cannot upload the script — paste this on the VPS

```bash
sudo tee /etc/nginx/sites-available/nexlify-panel-3000 <<'EOF'
map $http_x_forwarded_proto $forwarded_proto {
    default $http_x_forwarded_proto;
    ""      $scheme;
}
upstream nexlify_panel {
    server 127.0.0.1:3000;
    keepalive 32;
}
server {
    listen 3000;
    listen [::]:3000;
    server_name nexlify.live;
    client_max_body_size 100m;
    location / {
        proxy_pass http://nexlify_panel;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $forwarded_proto;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Port $server_port;
    }
}
EOF
sudo ln -sf /etc/nginx/sites-available/nexlify-panel-3000 /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

## PM2 must listen on localhost only

In `.env`:

```
PANEL_BEHIND_NGINX=1
PORT=3000
```

Then:

```bash
pm2 restart nexlify --update-env
ss -tlnp | grep 3000
```

Expected:

- `nginx` on `0.0.0.0:3000`
- `node` on `127.0.0.1:3000` only (not `0.0.0.0:3000`)

## Verify

```bash
curl -s http://127.0.0.1:3000/api/health
curl -sI http://nexlify.live:3000/login
```

Both should **not** return the HTTPS-port 400 page.

## 400 — Request Header Or Cookie Too Large

Panel session + license cookies can exceed nginx defaults. On the VPS:

```bash
sudo bash scripts/vps-fix-400-port-3000.sh
```

Or add inside the `server { }` block for port 3000:

```nginx
large_client_header_buffers 8 64k;
client_header_buffer_size 32k;
```

Then `sudo nginx -t && sudo systemctl reload nginx`.

Also clear site cookies for `nexlify.live` in your browser (old large cookies may persist).

## 500 Internal Server Error (nginx HTML, ~186 bytes)

Usually **Next.js is not running on `127.0.0.1:3000`** or nginx proxy to upstream is misconfigured.

On the VPS from project root:

```bash
sudo bash scripts/vps-fix-500-panel.sh
```

Or manually:

```bash
curl -s http://127.0.0.1:3000/api/health   # must return JSON, not fail
pm2 logs nexlify --lines 50                 # if health fails
sudo bash scripts/vps-fix-400-port-3000.sh  # rewrite nginx proxy
pm2 restart nexlify --update-env
```

Ensure `.env` has `PANEL_BEHIND_NGINX=1` and PM2 shows `127.0.0.1:3000` (not only `0.0.0.0:3000` without nginx).

## panel.nexlify.live shows marketing site instead of IPTV panel

Nginx is fine; the **wrong Node app** is on port 3000 (license/marketing site, not this panel repo).

```bash
curl -s http://127.0.0.1:3000/ | head -c 400   # if you see "IPTV Panel Licenses" → wrong app
pm2 list
bash scripts/vps-split-panel-and-website.sh
```

Run **nexlify-panel** on `127.0.0.1:3000`, marketing on another port (e.g. 3002). Open **https://panel.nexlify.live/login**.
